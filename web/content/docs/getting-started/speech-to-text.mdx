SabbathCue supports two STT engines, and you can swap between them at runtime
from the settings dialog. They map cleanly to two different operating
modes: zero-marginal-cost local inference, or offload-to-the-cloud
streaming.

## Option 1: Vosk (local, default)

Vosk is an offline speech recognition toolkit that transcribes audio in
real time using open dictation — no constrained phrase grammar. The default
small English model covers everyday vocabulary well, and larger Vosk models
improve accuracy for specialized or uncommon terms.

To install the Vosk model:

1. Download a model from [alphacephei.com/vosk/models](https://alphacephei.com/vosk/models)
2. Place it in `models/vosk/` (the app expects `vosk-model-small-en-us` by default)
3. Or set `SABBATHCUE_VOSK_MODEL_DIR` to point to your model folder

The model is required before Vosk transcription starts. The app shows
model installation status in Settings → Speech Recognition.

## Option 2: Deepgram

Deepgram is a cloud speech-to-text service that streams audio over a
WebSocket. SabbathCue also has a REST fallback that re-uploads short windows if
the WebSocket disconnects.

### 1. Get an API key

Sign up at [deepgram.com](https://deepgram.com/) and copy your API key.

### 2. Add it to `.env`

Create a `.env` file in the project root:

```bash
DEEPGRAM_API_KEY=dg_your_key_here
```

Restart `bun run tauri dev` so the new key is picked up.

### 3. Switch to Deepgram in the app

Open Settings → STT and choose **Deepgram**. The status indicator shows
WebSocket health in real time; if it falls back to REST, you'll see that
labeled too.

## Switching engines at runtime

Both engines are wired into the same transcript stream, so you can switch
without restarting the session. The active translation, sermon context,
and queue state are preserved across the swap.

## Comparing the two

| | Vosk (local) | Deepgram (cloud) |
| --- | --- | --- |
| API cost | Free | Per-minute billing |
| Internet required | No, after the model is installed | Yes (WebSocket) |
| Recognition mode | Open dictation | Keyword-boosted streaming |
| Long-form accuracy | Good with larger models | Excellent |
| Punctuation | None (plain text) | Excellent |
| Works offline | Yes | No |

Accuracy depends heavily on the Vosk model size and audio quality —
larger models produce better transcriptions but use more memory and CPU.
Deepgram accuracy is bounded by the round-trip to its nearest region.
Pick whichever fits your install: Vosk for the no-cost offline path,
Deepgram for the lowest latency and best transcription quality on a
reliable network.
