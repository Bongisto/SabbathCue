SabbathCue supports four STT engines, and you can swap between them at
runtime from the settings dialog. They map to two operating modes:
zero-marginal-cost local recognition, or cloud streaming when you want a
managed STT service.

## Option 1: Vosk

Vosk is the default local speech recognition engine. Audio stays on the
operator machine and the worker emits transcript events into the same detection
pipeline as the cloud providers.

To install the local model for development:

```bash
bun run download:vosk
```

Release builds can bundle a self-contained worker:

```bash
bun run build:vosk-sidecar
```

The app checks the Vosk model, worker, and runtime from Settings -> Speech.

## Option 2: Deepgram

Deepgram is a cloud speech-to-text service that streams audio over a WebSocket.
SabbathCue also has a REST fallback that uploads short windows if the WebSocket
disconnects.

1. Open the [Deepgram Console](https://console.deepgram.com/), select your
   project, and go to **Settings -> API Keys**.
2. Create a key named `SabbathCue` using the least-privileged transcription
   role available and an expiration you can maintain.
3. Copy the secret immediately; it is displayed only once.
4. Open **Settings -> Speech Recognition**, choose **Deepgram**, paste and save
   the key, then select **Test key**.

The key is stored in the OS keychain.

## Option 3: Soniox

Soniox is another cloud STT option with multilingual live streaming support.

1. Open the [Soniox Console](https://console.soniox.com/), enter **My First
   Project**, and select **API Keys**.
2. Generate and copy a new key.
3. Open **Settings -> Speech Recognition**, choose **Soniox**, paste and save
   the key, then select **Test key**.

The key is stored in the OS keychain.

## Option 4: Speechmatics

Speechmatics provides multilingual real-time partial and final transcripts.

1. Create an account or sign in to the
   [Speechmatics Portal](https://portal.speechmatics.com/).
2. Open **API Keys** from the project or account navigation and generate a key
   named `SabbathCue`.
3. Open **Settings -> Speech Recognition**, choose **Speechmatics**, paste and
   save the key, then select **Test key**.
4. Select the sermon language and start live transcription.

The long-lived key stays in the OS keychain and is used only by the native
desktop backend.

## Comparing Providers

|                     | Vosk (local)                       | Deepgram (cloud)          | Soniox (cloud)      | Speechmatics (cloud)       |
| ------------------- | ---------------------------------- | ------------------------- | ------------------- | -------------------------- |
| API cost            | Free                               | Provider rates apply      | Provider rates apply| Provider rates apply       |
| Internet required   | No                                 | Yes                       | Yes                 | Yes                        |
| Recognition mode    | Local worker + constrained grammar | Keyword-boosted streaming | Live streaming      | Partial + final streaming  |
| Transcript coverage | Best for verse/reference cues      | Full general speech       | Full general speech | Full multilingual speech   |
| Works offline       | Yes                                | No                        | No                  | No                         |

Pick Vosk for the no-cost offline path. Pick Deepgram, Soniox, or Speechmatics
when you have a reliable network and want cloud transcription quality. Use the
in-app current-pricing links before committing to a provider.
